# 文档骨架模板 V3.0

整份业务逻辑清单的章节骨架。支持 **正式版** 和 **草案版** 两种模式。

V3.0 变更：功能用例表6列（十字法），数据来源4列（溯源三问），新增角色泳道图、核心单据流转、系统交互时序、边界条件清单4个全局章节。

---

## 三、状态标记规范

### 草案标记

| 标记 | 使用位置 | 含义 |
|------|----------|------|
| `[草案]` | 预期结果、异常分支、状态流转、登录拦截、全局章节 | 设计意图，未验证 |
| `[待原型]` | 页面标题（H3） | 页面文件名待创建 |
| `[待确认]` | 数据来源（从哪来/往哪去）、业务问题 | 需用户决策 |
| `[必验]` | 假设清单、业务逻辑增强 | 关键假设，必须验证否则流程失败 |
| `[风险]` | 假设清单、业务逻辑增强 | 可能失败的假设，需关注 |
| `[暂略]` | 假设清单、Not Doing 清单 | 当前版本有意排除 |

### 正式版替换规则

从草案转换为正式版时：

| 草案标记 | 正式版替换 |
|----------|-----------|
| `[草案]` | 移除，替换为源码实现的具体描述（含颜色值、动画名） |
| `[待原型]` | 移除，页面文件名已存在 |
| `[待确认]` | 替换为确认结果，或保留建议答案格式 `（建议：...）` |
| `[必验]`/`[风险]` | 移至业务逻辑增强表，标注验证结果 |
| `[暂略]` | 移至 Not Doing 清单章节或直接移除 |

## 四、骨架使用说明

### 正式版

1. 模块数量和页面数量根据版本范围动态调整
2. 每个页面固定包含功能用例表（6列） + 关键字段数据来源（4列溯源三问）；截图为可选项（默认不加，仅用户要求时添加）
3. 业务逻辑增强表为可选，有业务疑问时才生成
4. 全局章节固定在文档末尾，按顺序：页面导航 → 状态流转 → 角色泳道图 → 核心单据流转 → 系统交互时序 → 边界条件清单
5. 二级标题之间用 `---` 分隔
6. 截图（仅用户要求时）使用相对路径 `./screenshots/{filename}.png`
7. 角色泳道图在涉及 ≥ 2 个角色时生成
8. 核心单据流转在有核心实体（订单、审批等）时生成
9. 系统交互时序在涉及跨系统交互时生成
10. 边界条件清单为必选章节

### 草案版

1. 必须包含 Not Doing 清单（≥ Doing 的 50%）
2. 每个页面必须包含假设清单（≥ 2 条假设）
3. 假设必须区分 [必验]/[风险]/[暂略]
4. 功能用例 6 列完整，异常分支标注 [草案]
5. 关键字段数据来源使用 4 列溯源三问格式
6. 不确定数据来源标注 [待确认]
7. 跳转关系和状态流转已定义（标注 [草案]）
8. 无截图，页面标题标注 [待原型]
9. 角色泳道图、系统交互时序、边界条件清单为必选章节
10. 核心单据流转在有核心实体时生成

| 模式 | 文件名 |
|------|--------|
| 正式版 | `业务逻辑清单_V{版本号}.md` |
| 草案版 | `业务逻辑清单_V{版本号}-草案.md` |

# 功能用例生成规则 V2.0

定义如何从 HTML/JS 源码中提取功能用例，基于十字法生成六列表格（场景/前置条件/操作步骤/预期结果/异常分支）。

---

## 核心原则

**每条用例必须能在源码中找到对应的实现。** 不臆测未实现的功能，不遗漏已实现的交互。

**十字法驱动：** 每个功能从横纵两个维度展开——横向追问角色×场景，纵向覆盖正常×异常。

---

## 十字法提取框架

### 横向：角色 × 场景（Who / When / What / Result）

对每个可交互元素追问四个维度：

| 维度 | 追问内容 | 映射到用例表列 |
|------|----------|----------------|
| **Who** | 哪个角色在操作？（用户/运营/系统） | 场景列中体现角色上下文 |
| **When/Where** | 在什么前置条件下操作？（已登录/已选商品/列表非空） | 前置条件列 |
| **What** | 做什么具体操作？ | 操作步骤列 |
| **Result** | 正常结果是什么？ | 预期结果列 |

### 纵向：正常 × 异常

对每条正常路径追问：

| 追问维度 | 具体问题 | 映射到用例表列 |
|----------|----------|----------------|
| **边界条件** | 为空、超限、重复、并发时怎么处理？ | 异常分支列 |
| **失败反馈** | 失败后用户看到什么？（Toast/弹窗/页面跳转/状态变化） | 异常分支列 |
| **数据回滚** | 失败后数据如何恢复？ | 异常分支列 |

### 异常场景分类

| 异常类型 | 提取方式（Extract） | 推演方式（Draft） |
|----------|---------------------|-------------------|
| 校验失败 | `if (!value)` / `maxlength` / `pattern` | 校验规则设计 |
| 边界超限 | 数量=1时禁用 / 金额超限 | 上下界设计 |
| 数据为空 | `if (data.length === 0)` | 空状态场景设计 |
| 重复操作 | 已收藏→再次点击→取消 | 重复操作处理设计 |
| 并发冲突 | 防重提交代码 / loading 状态 | 并发场景设计 |
| 网络异常 | `catch(err)` / 超时处理 | 网络异常提示设计 |

## 提取方法（Extract 模式）

### 步骤一：扫描 HTML 结构，识别可交互元素

逐行阅读 HTML，按以下模式识别：

| HTML 模式 | 提取的场景 |
|-----------|-----------|
| `<button>` 或 `.btn-*` | 按钮点击行为 |
| `<input>` / `<textarea>` | 表单输入、校验 |
| `<a href="...">` | 链接跳转 |
| `.tab-*` / Tab 切换结构 | Tab 筛选/切换 |
| `.filter-*` / 筛选标签 | 筛选过滤 |
| 弹窗结构（overlay + popup） | 弹窗打开/关闭 |
| `.badge` / 角标 | 角标数量 |
| 左滑/滑动结构 | 滑动手势操作 |
| 开关（toggle/switch） | 状态切换 |
| 星级评分组件 | 评分选择 |
| 图片上传区域 | 图片上传/删除 |
| 复选框/勾选框 | 勾选/取消勾选 |

### 步骤二：阅读 JS 事件监听，提取操作和触发条件

在 `<script>` 块中搜索以下模式：

| JS 模式 | 提取的操作 |
|---------|-----------|
| `addEventListener('click', fn)` | 点击操作及结果 |
| `addEventListener('input', fn)` | 输入操作及实时反馈 |
| `addEventListener('submit', fn)` | 表单提交及校验 |
| `onclick="..."` | 内联点击事件 |
| `location.href = 'xxx.html'` | 页面跳转 |
| `history.back()` | 返回导航 |
| `alert('...')` | 弹窗提示文案 |
| `classList.add/remove/toggle` | 样式状态切换 |

### 步骤三：跟踪条件分支，提取多场景用例

在 JS 中遇到 `if/else` 分支时，每个分支生成一条独立用例：

```
// 示例代码：数量控制
if (count === 1) {
  decreaseBtn.disabled = true;
} else {
  count--;
  updateTotal();
}
```

生成两条用例：
| 场景 | 操作 | 预期结果 |
|------|------|----------|
| 减少数量（最小值） | 点击-按钮 | 按钮置灰（`disabled`），数量不变 |
| 减少数量（正常） | 点击-按钮 | 数量-1，总计更新 |

### 步骤四：从 CSS 提取视觉反馈描述

在 `<style>` 块或 CSS 文件中查找：

| CSS 模式 | 提取的预期结果 |
|----------|--------------|
| `.active` 样式定义 | 选中态描述（颜色值、背景渐变） |
| `@keyframes` 动画名称 | 动画效果（缩放、渐变、滑动方向和时长） |
| `transition` 属性 | 过渡效果描述 |
| `display: none/flex` 切换 | 显示/隐藏行为 |
| `transform: scale(...)` | 缩放效果 |
| 颜色值（`#ff6034`） | 具体颜色描述，如"橙色（`#ff6034`）" |

### 步骤五：从数据结构提取展示类用例

JS 中的数据对象定义（如 `var items = [...]`）暗示展示用例：

```
// 示例：列表数据结构
var listItems = [
  { status: 'pending', statusText: '待处理', statusColor: '#ff9800', buttons: ['取消', '处理'] },
  { status: 'done', statusText: '已完成', statusColor: '#4caf50', buttons: ['查看'] },
];
```

从中提取：
- 状态颜色用例（不同状态对应不同颜色）
- 操作按钮用例（不同状态显示不同按钮组合）
- 空状态用例（数组为空时的展示）

## 用例编写规范（六列表格）

### 场景列

- 简短名词短语，2-6 字
- 不重复操作列的内容
- 同类场景可合并，用"/"分隔（如"单选/取消"、"全选/取消全选"）

**示例**：图片轮播、选择规格、提交订单、Tab筛选、空状态

### 前置条件列

- 描述触发该用例前必须满足的条件
- 可包含：用户状态（已登录/未登录）、页面状态（列表非空/购物车有商品）、数据条件（已选规格）
- 无特殊前置条件时填写"—"

**示例**：
- 已登录、已选择商品
- 购物车非空
- —（无特殊前置条件）

### 操作步骤列

- 简短名词短语，2-6 字
- 不重复操作列的内容
- 同类场景可合并，用"/"分隔（如"单选/取消"、"全选/取消全选"）

**示例**：图片轮播、选择规格、提交订单、Tab筛选、空状态

### 操作列

- 以动词开头，描述用户的具体动作
- 包含交互元素的名称或文案（用引号包裹按钮文案）
- 必要时包含前置条件

**示例**：
- 点击"已选规格"
- 左右滑动主图
- 点击"结算(N)"

### 预期结果列

- 具体到可验证的程度
- 包含：文案变化、颜色值、动画名称、页面跳转目标（附文件名）
- 不使用模糊描述（如"显示正确""正常工作"）

**好的预期结果**：
```
选中项高亮，对应的商品主图变更
```
```
跳转支付页（`payment.html`）
```
```
橙色渐变背景白字（`#ff6034` → `#ee0a24`）
```

**不好的预期结果**：
```
样式改变
```
```
跳转到下一个页面
```

### 异常分支列

- 描述该操作可能触发的异常路径及处理方式
- 格式：`异常条件 → 处理方式`（用箭头连接）
- 如果无异常分支，填写"—"
- Extract 模式：从源码 if/else、try/catch、错误回调中提取
- Draft 模式：基于业务场景推演，标注 `[草案]`

**好的异常分支**：
```
库存不足 → Toast"库存不足"并刷新商品信息
```
```
未勾选协议 → Toast"请阅读并同意用户协议"
```
```
SKU已下架 → 置灰不可选，显示"已售罄"
```

**不好的异常分支**：
```
失败
```
```
提示错误
```
```
不处理
```

## 特殊场景处理

### 空状态

当源码中有空状态判断逻辑（如 `if (data.length === 0)`）时，生成空状态用例。空状态用例的前置条件列填写"列表为空/数据为空"，异常分支列填"—"。

### 边界条件

从源码中的校验逻辑提取边界用例，**异常分支列是边界条件的主要承载列**：
- 数量最小值（qty=1 时 - 按钮置灰）→ 异常分支：`数量=1 → -按钮置灰disabled`
- 字符数上限（maxlength 属性）→ 异常分支：`超出上限 → 截断或显示字数提示`
- 重复操作（已收藏 → 再次点击 → 取消收藏）→ 异常分支：`已收藏状态 → 点击变为取消收藏`

### 动画效果

从 CSS `@keyframes` 提取，用括号标注动画名和参数，写入预期结果列：
```
绿色勾选图标缩放动画（scaleIn：0→1.1→1，0.4s）
```

### 多状态展示

当页面支持多种状态（如订单的不同状态），为每种状态各生成一条用例，预期结果包含对应颜色值和操作按钮。

### 异常分支提取（十字法纵向）

从源码中提取异常分支的代码模式：

| 代码模式 | 提取的异常分支 |
|----------|----------------|
| `if (!value)` 或 `value === ''` | 校验失败 → Toast提示必填 |
| `if (count <= 0)` 或 `count > max` | 边界超限 → 禁用操作或提示 |
| `try { ... } catch(e) { ... }` | 网络异常 → 错误提示 |
| `if (err.code === 'XXX')` | 服务端错误码 → 对应提示 |
| `if (res.success === false)` | 接口失败 → Toast错误信息 |
| `if (loading)` 或 `btn.disabled` | 防重复提交 → 按钮禁用 |

## 常见错误

| 错误 | 正确做法 |
|------|----------|
| 凭空编造源码中不存在的功能 | 每条用例必须能在源码中找到对应实现 |
| 预期结果写"显示正确" | 具体到文案、颜色、动画、跳转目标 |
| 把多个操作合并为一条用例 | 每个独立交互一条用例 |
| 遗漏空状态和边界条件 | 检查 if/else 分支和校验代码 |
| 场景列写完整的句子 | 2-6字名词短语 |
| 操作列遗漏按钮文案 | 用引号包裹按钮/链接文案 |
| 前置条件列写操作描述 | 仅写触发条件（状态、数据条件），不写操作 |
| 异常分支列只写"失败"或"报错" | 具体到异常类型+处理方式（如`库存不足→Toast提示`） |
| 正常用例不写异常分支 | 即使无异常也显式填写"—"，明确已分析过 |
| 跳过十字法纵向追问 | 每条正常用例必须追问边界和异常场景 |

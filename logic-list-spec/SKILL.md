# 业务逻辑清单生成技能 V0.22

> 版本管理见 [VERSIONING.md](VERSIONING.md)

双模式支持：**Draft模式**（需求正向生成草案）+ **Extract模式**（源码逆向提取）。融合十字法、泳道角色法、状态机法、数据溯源三问等方法论。

---

## 双模式说明

| 模式 | 输入来源 | 适用场景 | 输出文件 |
|------|----------|----------|----------|
| **Draft** | 需求描述 + codebase扫描 | 无原型页面，首次设计 | `业务逻辑清单_V{版本}-草案.md` |
| **Extract** | HTML源码逆向提取 | 有原型页面 | `业务逻辑清单_V{版本}.md` |

---

## 分析方法论

本技能融合以下 5 种结构化分析方法（详见 [rules/business-analysis.md](rules/business-analysis.md)）：

| 方法 | 核心思路 | 输出 |
|------|----------|------|
| **十字法** | 横向追问角色×场景（Who/When/What/Result），纵向追问正常×异常 | 6列功能用例表（+前置条件+异常分支） |
| **泳道角色法** | 按角色×阶段列交互节点，区分用户操作和系统判断 | 角色泳道图矩阵表 |
| **状态机法** | 列出所有状态→画跳转路径→标注触发条件和守卫条件 | 状态流转表 + stateDiagram |
| **数据溯源三问** | 逐字段追问：从哪来/往哪去/变了怎么办 | 4列数据来源表 |
| **系统交互时序** | 画出请求→响应链路，标注同步/异步和失败策略 | 系统交互时序表 + sequenceDiagram |

**6步落地套路：**

1. 画角色泳道图 → 理清有哪些人、各做什么
2. 拆功能用例 → 逐功能展开正常和异常场景（十字法）
3. 画状态机 → 有状态的对象单独建模型
4. 做字段溯源 → 每个关键页面的字段过一遍（三问）
5. 串系统时序 → 跨系统的请求链路走通
6. 补充边界条件 → 并发、幂等、数据一致性

## 模式自动检测规则

1. **用户指定** `--mode=draft` 或 `--mode=extract`
2. **检测页面存在**：`page-index.md` 中页面文件存在 → Extract模式
3. **用户口述**："没有原型"/"首次设计"/"新功能"/"草案" → Draft模式
4. **默认**：Extract模式

## 一、Draft模式流程

适用于无原型HTML的首次设计场景，深度融合 idea-refine 方法论和6步落地套路。

```
需求输入 → 需求收集 → 页面收敛 → 6步分析 → 输出草案 → 用户确认
```

| 阶段 | 目标 | 详细规则 |
|------|------|---------|
| 一、需求收集 | HMW重述 → 澄清问题 → 变体生成 → 上下文扫描 | [rules/requirement-collection.md](rules/requirement-collection.md) |
| 二、页面收敛 | 聚类页面 → 压力测试 → 显性假设 → Not Doing | [rules/draft-generation.md](rules/draft-generation.md) |
| 三、6步分析 | 泳道 → 用例(十字法) → 状态机 → 字段溯源 → 时序 → 边界 | [rules/draft-generation.md](rules/draft-generation.md) + [rules/business-analysis.md](rules/business-analysis.md) |
| 四、输出草案 | 按骨架模板组装完整文档 | [templates/doc-skeleton.md](templates/doc-skeleton.md) |

**关键约束：**
- 不进入阶段二，直到用户画像 + 成功标准已明确
- 每页至少 2 条假设，标注 `[必验]`/`[风险]`/`[暂略]`
- Not Doing 清单项 ≥ Doing 项的 50%
- 总页面数 ≤ 10
- 功能用例表必须 6 列（场景/前置条件/操作步骤/预期结果/异常分支）
- 数据来源必须 4 列溯源三问（字段/从哪来/往哪去/变了怎么办）

**输出：** `doc/V{版本}/业务逻辑清单_V{版本}-草案.md`，无截图，页面标注 `[待原型]`

## 二、Extract模式流程（原有流程）

适用于有原型HTML的验证场景。

### 阶段一：材料收集与范围确认

1. **读取导航索引** — `page-index.md` 提取带版本标记的页面列表
2. **读取现有文档** — 避免重复/识别增量
3. **读取页面源码** — 按 `reference/material-checklist.md` 逐页读取
4. **制定文档计划** — 提交用户审阅

### 阶段二：逐页生成文档内容

5. **逐页生成三表**（十字法提取）：
   - 功能用例表（6列：场景/前置条件/操作步骤/预期结果/异常分支）
   - 关键字段数据来源（4列溯源三问：字段/从哪来/往哪去/变了怎么办）
   - 业务逻辑增强（可选）
6. **生成全局章节**：
   - 跳转关系表
   - 返回导航
   - 登录拦截
   - 状态流转表（状态机法，有状态实体时必选）
   - 角色泳道图（≥2个角色时必选）
   - 核心单据流转（有核心实体时必选）
   - 系统交互时序（跨系统时必选）
   - 边界条件清单（必选）

### 阶段三：渲染与输出

> **截图默认关闭**：除非用户主动要求添加截图（如"加截图"、"截图"、"生成截图"或传入 `--screenshot`），否则**不生成、不引用截图**。默认产出纯逻辑文档，避免截图收集阻塞交付。

7. **（可选）自动截图** — 仅当用户要求时执行 `scripts/screenshot.py` 为每个页面截图
8. **流程图渲染** — 将 mermaid 流程图渲染为 SVG（详见 `rules/flowchart-rules.md` 的渲染方案）
9. **合并输出** — 输出完整文档；若已生成截图则插入路径，否则不插入任何截图占位

## 三、输入参数

| 参数名称 | 参数说明 | 必填 |
|----------|----------|------|
| 模式 | `--mode=draft` 或 `--mode=extract` | 否（自动检测） |

| 截图 | `--screenshot` 或用户主动要求 | 否（**默认关闭**） |

## 四、输出规范

### 输出文件

| 模式 | 文件名 | 截图 |
|------|--------|------|
| Draft | `业务逻辑清单_V{版本}-草案.md` | 无 |
| Extract | `业务逻辑清单_V{版本}.md` | 无（用户主动要求时才加 `screenshots/*.png`） |

### 文档章节结构

```
业务逻辑清单 V{版本号}
├── 文档标题 + 版本概述
├── 一～{N-1}、{模块名}（每模块含多个页面）
│   └── 每页：[截图-可选] + 功能用例表(6列) + 数据来源(4列溯源) + 业务增强(可选)
├── {N}、页面导航（跳转关系 + 返回导航 + 登录拦截）
├── {N+1}、状态流转（可选，有状态实体时）
├── {N+2}、角色泳道图（≥2角色时必选）
├── {N+3}、核心单据流转（有核心实体时必选）
├── {N+4}、系统交互时序（跨系统时必选）
└── {N+5}、边界条件清单（必选）
```

### 草案→正式版转换规则

| 草案标记 | 正式版替换 |
|----------|-----------|
| `[草案]` | 移除，替换为源码实现描述 |
| `[待原型]` | 移除，页面文件已存在 |
| `[待确认]` | 替换为确认结果或保留建议格式 |
| `[必验]`/`[风险]` | 标注验证结果 |

## 六、Anti-patterns（反模式）

| 反模式 | 正确做法 |
|--------|----------|
| Draft模式不标注假设 | 每页必须标注假设清单（≥2条） |
| Draft模式无Not Doing | Not Doing ≥ Doing的50% |
| Draft模式预期结果模糊 | 具体到交互元素、颜色、页面名 |
| Draft模式跳过codebase扫描 | 必须扫描现有组件和约束 |
| Extract模式臆测未实现功能 | 每条用例必须在源码有对应 |
| 模式判断错误 | 优先用户指定，其次检测页面存在 |
| 跳过十字法纵向追问 | 每条正常用例必须有异常分支（或显式"—"） |
| 数据来源只写"从哪来" | 三问缺一不可，尤其是"变了怎么办" |
| 状态机只画正常路径 | 每个状态必须覆盖超时和异常路径 |
| 系统时序不标注同步/异步 | 每条链路必须标注 |
| 边界条件只写"需处理" | 必须写明具体处理策略 |

## 七、验证清单

**Draft模式验证：**

- [ ] HMW问题陈述已写入头部
- [ ] 成功标准已定义
- [ ] Not Doing清单已列出（≥Doing的50%）
- [ ] 每个页面有假设清单（≥2条）
- [ ] 假设已区分 [必验]/[风险]/[暂略]
- [ ] 功能用例6列完整（场景/前置条件/操作步骤/预期结果/异常分支）
- [ ] 功能用例预期结果具体且标注 [草案]
- [ ] 关键字段数据来源4列溯源三问完整
- [ ] 跳转关系和状态流转已定义
- [ ] 角色泳道图已生成（≥2角色时）
- [ ] 系统交互时序已画出（标注同步/异步）
- [ ] 边界条件清单已补充（并发/幂等/一致性）

**Extract模式验证：**

- [ ] 页面范围与版本标记一致
- [ ] 每条用例能在源码找到对应实现
- [ ] 功能用例6列完整，异常分支从源码提取
- [ ] 关键字段数据来源4列溯源三问完整
- [ ] 截图（仅用户要求时生成并插入相对路径）
- [ ] 跳转关系完整
- [ ] 标题编号连续
- [ ] 边界条件清单已补充

# 输入验证规则 V2.0

定义如何验证用户输入（口述模式和草案模式）是否符合原型生成要求。

V2.0 变更：草案模式验证适配 logic-list-spec V0.22 的 6 列用例表（+前置条件+异常分支）和 4 列溯源三问字段表。

---

## 验证层次

| 层次 | 验证对象 | 验证目的 |
|------|----------|----------|
| **模式验证** | 输入类型 | 确定口述/草案模式 |
| **结构验证** | 输入内容 | 确保必要信息完整 |
| **语义验证** | 关键词含义 | 确保可映射到规范 |
| **一致性验证** | 内容关联 | 确保页面/用例/字段关联正确 |

## 口述模式输入验证

### 验证规则

| 验证项 | 验证规则 | 验证方式 | 失败处理 |
|--------|----------|----------|----------|
| **最小信息量** | 包含至少页面类型+用户角色 | 关键词提取 | 提示补充信息："请明确页面用途和用户角色" |
| **关键词可提取** | 至少提取一个一级关键词 | 关键词匹配 | 提示重新描述："无法理解页面用途，请重新描述" |
| **规范可匹配** | 用户角色+页面类型可确定规范 | 规范推断矩阵 | 提示选择规范："请选择页面类型：后台管理/小程序" |
| **模板可匹配** | 页面类型可匹配到模板 | 模板推断矩阵 | 提示补充："未找到匹配模板，请补充页面功能描述" |

### 口述模式验证清单

- [ ] 输入包含页面类型关键词
- [ ] 输入包含用户角色关键词
- [ ] 关键词可提取（至少1个一级关键词）
- [ ] 规范类型可确定（后台/小程序）
- [ ] 模板类型可匹配

### 口述模式验证示例

**输入：**
```
设计一个订单列表页面，运营人员可以查看所有订单
```

**验证结果：**
```json
{
  "valid": true,
  "checks": [
    {"item": "页面类型关键词", "result": "pass", "value": "订单列表"},
    {"item": "用户角色关键词", "result": "pass", "value": "运营人员"},
    {"item": "关键词可提取", "result": "pass", "count": 2},
    {"item": "规范可匹配", "result": "pass", "spec": "后台规范"},
    {"item": "模板可匹配", "result": "pass", "template": "表格列表页"}
  ]
}
```

## 草案模式输入验证

### 验证规则

| 验证项 | 验证规则 | 验证方式 | 失败处理 |
|--------|----------|----------|----------|
| **结构完整** | 包含必要章节结构 | 章节检测 | 提示补充："草案缺少必要章节：{缺失章节列表}" |
| **HMW陈述存在** | 包含HMW问题陈述 | HMW关键词检测 | 提示补充："请补充HMW问题陈述" |
| **页面章节存在** | 至少包含一个H3页面章节 | H3标题检测 | 提示补充："草案缺少页面定义" |
| **用例表非空** | 功能用例表至少包含1条用例 | 表格行数检测 | 提示补充："功能用例表为空，请补充用例" |
| **用例表结构正确** | 功能用例表包含必要列 | 表头检测 | 提示补充："功能用例表缺少必要列：{缺失列}" |
| **字段表非空** | 关键字段数据来源表至少包含1条 | 表格行数检测 | 提示补充："关键字段数据来源表为空" |
| **跳转关系存在** | 页面导航跳转关系定义存在 | 跳转关系表检测 | 提示补充："缺少页面跳转关系定义" |

### 草案必要章节清单

| 章节名称 | 必要性 | 检测规则 |
|----------|--------|----------|
| HMW问题陈述 | **必要** | 包含 `HMW` 关键字或 `How Might We` |
| 成功标准 | 推荐 | 包含 `成功标准` 章节标识 |
| Not Doing清单 | **必要（草案）** | 包含 `Not Doing` 章节标识 |
| 页面章节（H3） | **必要** | 至少包含一个H3页面标题 |
| 功能用例表 | **必要** | 每个页面章节包含功能用例表 |
| 关键字段数据来源 | **必要** | 每个页面章节包含字段表 |
| 页面导航（跳转关系） | **必要** | 包含跳转关系表 |

### 草案功能用例表结构验证（V0.22 六列）

| 列 | 检测规则 | 必要性 | 缺失处理 |
|--------|----------|--------|----------|
| # | 表头包含 `#` 或 `序号` | 可选 | 跳过序号，使用默认递增 |
| 场景 | 表头包含 `场景` | **必要** | 提示补充场景列 |
| 前置条件 | 表头包含 `前置条件` | 推荐 | 提示补充前置条件列（影响条件渲染） |
| 操作步骤 | 表头包含 `操作步骤` 或 `操作` | **必要** | 提示补充操作列 |
| 预期结果 | 表头包含 `预期结果` 或 `预期` | **必要** | 提示补充预期结果列 |
| 异常分支 | 表头包含 `异常分支` 或 `异常` | 推荐 | 提示补充异常分支列（影响错误组件生成） |

**兼容性说明：** 旧版 4 列格式（场景/操作/预期结果）仍视为有效（3 个必要列齐全），但会提示建议升级到 6 列以获得前置条件和异常分支的组件映射。

### 草案关键字段数据来源表结构验证（V0.22 四列溯源三问）

| 列 | 检测规则 | 必要性 | 缺失处理 |
|--------|----------|--------|----------|
| 字段 | 表头包含 `字段` | **必要** | 提示补充字段列 |
| 从哪来 | 表头包含 `从哪来` 或 `数据来源` | **必要** | 提示补充来源列 |
| 往哪去 | 表头包含 `往哪去` | 推荐 | 提示补充去向列（影响提交绑定） |
| 变了怎么办 | 表头包含 `变了怎么办` | 推荐 | 提示补充变更策略列（影响校验逻辑） |

**兼容性说明：** 旧版 2 列格式（字段/数据来源）仍视为有效（2 个必要列齐全），但会提示建议升级到 4 列溯源三问。

### 草案模式验证清单

- [ ] HMW问题陈述存在
- [ ] Not Doing清单存在
- [ ] 至少包含一个H3页面章节
- [ ] 每个页面包含功能用例表
- [ ] 每个功能用例表结构正确（场景/前置条件/操作步骤/预期结果/异常分支）
- [ ] 每个功能用例表非空（至少1条）
- [ ] 每个页面包含关键字段数据来源表
- [ ] 每个字段表结构正确（字段/从哪来/往哪去/变了怎么办）
- [ ] 页面导航跳转关系表存在
- [ ] 状态流转表存在（可选，有状态流转时必要）

## 验证失败处理流程

### 处理策略

| 验证结果 | 处理方式 |
|----------|----------|
| **完全通过** | 继续执行生成流程 |
| **部分通过** | 提示缺失项，用户确认后继续（使用默认值/占位符） |
| **验证失败** | 提示补充必要信息，暂停生成 |

### 缺失项默认处理

| 缺失项 | 默认处理 | 占位符示例 |
|--------|----------|------------|
| 关键字段数据来源表 | 使用空占位符 | `<div class="field-placeholder">[待补充字段数据来源]</div>` |
| 状态流转表 | 不生成状态逻辑 | 无状态交互代码 |
| 跳转关系表 | 不生成跳转逻辑 | 无跳转链接代码 |
| 某个字段来源 | 使用占位符展示 | `[待确认]` 文本 |

## 验证输出格式

```json
{
  "mode": "oral|draft",
  "valid": true|false,
  "checks": [
    {
      "item": "验证项名称",
      "result": "pass|fail",
      "value": "提取值（可选）",
      "message": "失败原因（可选）"
    }
  ],
  "missing": ["缺失项列表"],
  "suggestion": "补充建议",
  "can_proceed": true|false
}
```

## 验证时机

| 时机 | 验证内容 |
|------|----------|
| **输入接收时** | 模式识别 + 结构完整性 |
| **解析前** | 章节完整性 |
| **生成前** | 映射可行性（关键词/模板可匹配） |

## 验证工具函数

```javascript
// 口述模式验证
function validateOralInput(input) {
  const keywords = extractKeywords(input);
  const checks = [];
  
  checks.push({
    item: "页面类型关键词",
    result: keywords.page_type ? "pass" : "fail",
    value: keywords.page_type
  });
  
  checks.push({
    item: "用户角色关键词",
    result: keywords.user_role ? "pass" : "fail",
    value: keywords.user_role
  });
  
  const spec = inferSpec(keywords);
  checks.push({
    item: "规范可匹配",
    result: spec ? "pass" : "fail",
    value: spec
  });
  
  const template = inferTemplate(keywords);
  checks.push({
    item: "模板可匹配",
    result: template ? "pass" : "fail",
    value: template
  });
  
  return {
    mode: "oral",
    valid: checks.every(c => c.result === "pass"),
    checks,
    can_proceed: checks.every(c => c.result === "pass")
  };
}

// 草案模式验证
function validateDraftInput(input) {
  const checks = [];
  
  checks.push({
    item: "HMW问题陈述",
    result: input.includes("HMW") || input.includes("How Might We") ? "pass" : "fail"
  });
  
  checks.push({
    item: "Not Doing清单",
    result: input.includes("Not Doing") ? "pass" : "fail"
  });
  
  const h3Count = (input.match(/^###\s/gm) || []).length;
  checks.push({
    item: "H3页面章节",
    result: h3Count > 0 ? "pass" : "fail",
    value: h3Count
  });
  
  return {
    mode: "draft",
    valid: checks.every(c => c.result === "pass"),
    checks,
    can_proceed: checks.filter(c => c.result === "fail").length < 2
  };
}
```

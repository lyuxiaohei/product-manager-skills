# 输出HTML验证规则

定义如何验证生成的HTML文件是否符合原型设计规范。

---

## 验证层次

| 层次 | 验证对象 | 验证目的 |
|------|----------|----------|
| **结构验证** | HTML结构 | 确保页面框架正确 |
| **规范验证** | 设计规范 | 确保符合后台/小程序规范 |
| **功能验证** | 功能完整性 | 确保所有用例/字段已映射 |
| **运行验证** | 可运行性 | 确保可直接浏览器运行 |

---

## 通用验证规则

| 验证项 | 验证规则 | 验证方式 | 失败处理 |
|--------|----------|----------|----------|
| **零依赖** | 无外部CSS/JS文件引用 | 检查 `<link>` 和 `<script src>` | 移除外部引用，改为内联 |
| **内联完整** | CSS/JS内联在 `<style>` 和 `<script>` 内 | 检查 `<style>` 和 `<script>` 标签 | 修正内联位置 |
| **可运行** | 直接浏览器打开可正常运行 | 实际浏览器测试 | 修正运行错误 |
| **无Emoji** | 无Emoji字符使用 | 文本内容检测 | 替换为SVG图标或纯文本 |
| **HTML5语法** | 使用HTML5语义化标签 | 标签检测 | 修正为HTML5标签 |
| **DOCTYPE声明** | 包含 `<!DOCTYPE html>` | 文件开头检测 | 添加DOCTYPE声明 |
| **charset声明** | 包含 `<meta charset="UTF-8">` | head区域检测 | 添加charset声明 |

### 通用验证清单

- [ ] 文件包含DOCTYPE声明
- [ ] 文件包含charset声明
- [ ] 无外部CSS文件引用
- [ ] 无外部JS文件引用
- [ ] CSS内联在 `<style>` 标签内
- [ ] JS内联在 `<script>` 标签内
- [ ] 无Emoji字符使用
- [ ] 使用HTML5语义化标签
- [ ] 可直接浏览器打开运行

## 验证工具函数

### CSS颜色验证

```javascript
function validateColor(cssText, expectedColor, property) {
  const regex = new RegExp(`${property}[^:]*:\\s*([^;]+)`);
  const match = cssText.match(regex);
  if (!match) return { valid: false, message: `缺少${property}属性` };
  
  const actualColor = match[1].trim().replace(/['"]/g, '');
  if (actualColor !== expectedColor && !actualColor.includes(expectedColor)) {
    return { valid: false, expected: expectedColor, actual: actualColor };
  }
  return { valid: true };
}
```

### 尺寸验证

```javascript
function validateSize(cssText, expectedSize, property) {
  const regex = new RegExp(`${property}[^:]*:\\s*([^;]+)`);
  const match = cssText.match(regex);
  if (!match) return { valid: false, message: `缺少${property}属性` };
  
  const actualSize = match[1].trim();
  if (actualSize !== expectedSize) {
    return { valid: false, expected: expectedSize, actual: actualSize };
  }
  return { valid: true };
}
```

### 结构验证

```javascript
function validateStructure(html, className) {
  const regex = new RegExp(`class="[^"]*${className}[^"]*"`);
  const hasStructure = regex.test(html);
  return { valid: hasStructure, message: hasStructure ? '存在' : `缺少${className}结构` };
}
```

### Emoji验证

```javascript
function validateNoEmoji(text) {
  const emojiRegex = /[\u{1F300}-\u{1F9FF}]|[\u{2600}-\u{26FF}]|[\u{2700}-\u{27BF}]/u;
  const hasEmoji = emojiRegex.test(text);
  return { valid: !hasEmoji, message: hasEmoji ? '包含Emoji' : '无Emoji' };
}
```

## 验证时机

| 时机 | 验证内容 |
|------|----------|
| **生成后即时验证** | 通用验证 + 规范验证 |
| **用户确认前** | 功能完整性验证（草案模式） |
| **交付前** | 运行验证（实际浏览器测试） |

## 验证通过标准

| 验证类型 | 通过标准 |
|----------|----------|
| **通用验证** | 所有通用验证项通过 |
| **规范验证** | 所有对应规范验证项通过 |
| **草案验证** | 核心验证项通过（允许非核心项缺失） |
| **可交付** | `can_ship = true`（无致命错误） |

# 口述关键词提取规则

定义如何从用户口述描述中提取关键信息，用于推断页面类型和选择组件。

---

## 关键词分类

### 一级关键词（必提取）

| 关键词类别 | 提取规则 | 权重 | 示例 |
|------------|----------|------|------|
| **页面类型** | 提取页面用途关键词 | 高 | 商品列表、订单管理、商品详情、购物车 |
| **用户角色** | 提取用户身份关键词 | 高 | 运营人员、用户、管理员、买家 |
| **核心操作** | 提取动词+名词组合 | 高 | 搜索商品、添加购物车、提交订单、查看详情 |

### 二级关键词（按需提取）

| 关键词类别 | 提取规则 | 权重 | 示例 |
|------------|----------|------|------|
| **交互方式** | 提取交互动词 | 中 | 点击、拖拽、滑动、筛选、切换 |
| **数据展示** | 提取数据相关词汇 | 中 | 列表、卡片、表格、分页、网格 |
| **页面元素** | 提取UI元素词汇 | 中 | 按钮、弹窗、导航、表单、输入框 |
| **业务场景** | 提取场景描述词 | 中 | 下单、支付、审批、审核、售后 |

---

## 关键词权重矩阵

| 权重级别 | 关键词类型 | 影响范围 | 决策权重 |
|----------|------------|----------|----------|
| **高权重** | 页面类型、用户角色 | 决定规范选择 | 60% |
| **中权重** | 核心操作、数据展示 | 决定组件选择 | 30% |
| **低权重** | 交互方式、页面元素 | 决定样式细节 | 10% |

---

## 提取算法

### Step 1: 分词与标注

```
输入文本 → 分词 → 词性标注 → 识别名词/动词/形容词
```

### Step 2: 关键词匹配

```
分词结果 → 匹配关键词词典 → 标记关键词类别 → 计算权重
```

### Step 3: 关键词排序

```
匹配结果 → 按权重排序 → 输出关键词列表
```

## 提取示例

### 示例1: 商品列表页（后台）

**输入：**
```
设计一个商品列表页面，运营人员可以查看所有商品，支持搜索、筛选、分页，点击可以查看详情
```

**提取结果：**
```json
{
  "page_type": "商品列表",
  "user_role": "运营人员",
  "core_operations": ["查看商品", "搜索", "筛选", "分页", "查看详情"],
  "data_display": ["列表"],
  "interactions": ["点击"],
  "spec_type": "backend",
  "template": "表格列表页",
  "components": ["BaseTable", "SearchInput", "FilterBar", "Pagination", "DetailModal"]
}
```

## 关键词缺失处理

| 缺失类型 | 处理方式 |
|----------|----------|
| 缺少页面类型 | 提示用户明确页面用途 |
| 缺少用户角色 | 默认推断：含"管理/审批"→后台，含"购物/下单"→小程序 |
| 缺少核心操作 | 生成基础展示页面，提示补充操作 |
| 关键词无法映射 | 提示用户重新描述或选择模板 |

---

## 口述模式输入验证

| 验证项 | 验证规则 | 失败处理 |
|--------|----------|----------|
| 最小信息量 | 包含至少页面类型+用户角色 | 提示补充信息 |
| 信息可理解 | 关键词可映射到规范/模板 | 提示重新描述 |
| 规范可匹配 | 用户角色+页面类型可确定规范 | 提示选择规范 |

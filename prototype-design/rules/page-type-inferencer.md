# 页面类型推断规则

定义如何基于关键词推断页面类型和适用规范（后台/小程序）。

---

## 推断流程

```
关键词数据 → 规范推断 → 模板推断 → 组件推断 → 页面配置
```

---

## 规范推断矩阵

### 基于用户角色推断

| 用户角色关键词 | 推断规范 | 页面特征 |
|----------------|----------|----------|
| 运营人员、运营、运营侧、管理员、管理、后台人员 | **后台规范** | PC端、左侧导航、表格布局、最小宽度1200px |
| 用户、买家、消费者、移动端、小程序、微信 | **小程序规范** | 移动端、手机框架375px、卡片布局、底部Tab栏 |

### 基于页面类型推断（辅助判断）

| 页面类型关键词 | 推断规范 | 判断依据 |
|----------------|----------|----------|
| 管理、审批、配置、素材库、编辑器、后台、运营 | **后台规范** | 运营工具类页面 |
| 详情、购物、支付、订单、评价、售后、用户 | **小程序规范** | 用户交互类页面 |

### 规范推断优先级

```
用户角色关键词 > 页面类型关键词 > 默认后台规范
```

## 复合页面处理

当关键词指向多个页面类型时：

### 处理策略

| 策略 | 适用场景 | 处理方式 |
|------|----------|----------|
| **优先匹配** | 关键词有明显权重差异 | 按权重优先选择模板 |
| **组合模板** | 多个模板可组合 | 组合多个模板核心组件 |
| **用户选择** | 无法确定唯一模板 | 输出候选模板供用户选择 |

### 组合示例

**输入：**
```
设计一个订单管理页面，运营可以查看订单列表，点击进入订单详情，支持审批操作
```

**分析：**
- 页面类型：订单管理 + 订单详情 + 审批 → 复合页面
- 推断规范：后台规范（运营角色）
- 组合方案：表格列表页（订单列表） + 详情页（订单详情审批）

**输出配置：**
```json
{
  "pages": [
    {"template": "表格列表页", "file": "order_list.html"},
    {"template": "审批详情页", "file": "order_detail.html"}
  ],
  "navigation": [
    {"from": "order_list", "trigger": "点击订单", "to": "order_detail"}
  ]
}
```

## 组件推断矩阵

### 基于核心操作推断

| 核心操作关键词 | 后台组件 | 小程序组件 |
|----------------|----------|------------|
| 搜索、查找、检索 | SearchInput + FilterBar | 搜索框 + 筛选栏 |
| 筛选、过滤 | FilterBar + StatusTag | 筛选下拉 + Tab |
| 分页、翻页 | Pagination | 滚动加载 |
| 查看详情 | DetailModal/Drawer | 详情页跳转 |
| 添加、新增 | AddButton + FormModal | 添加按钮 + 表单页 |
| 删除、移除 | DeleteButton | 左滑删除 |
| 编辑、修改 | EditButton + FormModal | 编辑入口 |
| 审批、审核 | ApprovalForm | 审批操作栏 |
| 上传、上传图片 | Upload | 图片上传区 |

### 基于数据展示推断

| 数据展示关键词 | 后台组件 | 小程序组件 |
|----------------|----------|------------|
| 列表、表格 | BaseTable | 卡片列表 |
| 网格、素材 | 5列MaterialGrid | 2列ProductGrid |
| 卡片 | InfoCard | Card组件 |
| 详情、信息 | InfoList | 信息展示区 |
| 时间线、进度 | Timeline | 时间线组件 |
| 图表、统计 | Chart | 数据卡片 |

## 推断失败处理

| 失败类型 | 处理方式 |
|----------|----------|
| 无法推断规范 | 默认后台规范，提示用户确认角色 |
| 无法推断模板 | 输出基础空白模板，提示补充页面类型 |
| 无法推断组件 | 生成基础组件，提示补充操作 |
| 关键词冲突 | 输出多个候选方案，提示用户选择 |

# 草案解析规则 V2.0

定义如何解析 logic-list-spec V0.22+ 生成的业务逻辑草案文档，提取关键信息用于原型生成。

V2.0 变更：适配 logic-list-spec V0.22 的 6 列功能用例表（+前置条件+异常分支）和 4 列溯源三问字段表。

---

## 输入来源

接收 `logic-list-spec` Draft 模式输出的草案文档，文件名格式：
- `业务逻辑清单_V{版本}-草案.md`

---

## 解析层次

草案文档按以下层次解析：

### 第一层：全局信息（必解析）

| 字段名称 | 字段标识 | 解析规则 | 输出用途 |
|----------|----------|----------|----------|
| HMW问题陈述 | `hmw_statement` | 检索 `**HMW 问题陈述**` 后的问句 | 页面定位依据 |
| 成功标准 | `success_criteria` | 检索 `**成功标准**` 后的列表项 | 验收清单生成 |
| Not Doing清单 | `not_doing` | 检索 `## Not Doing 清单` 后的表格 | 不生成项过滤 |

### 第二层：页面信息（逐页解析）

| 字段名称 | 字段标识 | 解析规则 | 输出用途 |
|----------|----------|----------|----------|
| 页面名称 | `page_name` | H3标题中的页面名称部分 | 文件命名 |
| 页面定位 | `page_positioning` | H3标题后的引用块内容 | 规范选择依据 |
| 压力测试结论 | `stress_test` | 检索 `User Value/Feasibility/Differentiation` | 页面重要性判断 |
| 假设清单 | `assumptions` | 检索 `假设清单` 后的列表项 | 验证提示生成 |
| 功能用例表 | `use_cases` | 检索 `功能用例表` 后的表格 | 组件实例化 |
| 关键字段数据来源 | `field_sources` | 检索 `关键字段数据来源` 后的表格 | UI元素映射 |
| 业务逻辑增强 | `business_logic` | 检索 `业务逻辑增强` 后的表格（可选） | 交互规则补充 |

### 第三层：全局流程（文档末尾）

| 字段名称 | 字段标识 | 解析规则 | 输出用途 |
|----------|----------|----------|----------|
| 跳转关系 | `navigation` | 检索 `跳转关系` 后的表格 | 跳转逻辑生成 |
| 返回导航 | `return_nav` | 检索 `返回导航` 后的说明 | 返回按钮实现 |
| 登录拦截 | `login_guard` | 检索 `登录拦截` 后的表格 | 登录检查实现 |
| 状态流转 | `state_flow` | 检索 `状态流转` 后的表格或mermaid | 交互逻辑生成 |

## 解析流程

### Phase 1: 全局信息提取

```
草案文档 → 检索HMW → 提取问题陈述 → 存入全局上下文
草案文档 → 检索成功标准 → 提取列表项 → 存入验收清单
草案文档 → 检索Not Doing → 提取表格内容 → 存入排除列表
```

### Phase 2: 逐页解析

```
草案文档 → 识别H3标题 → 提取页面名称 → 存入页面列表
每个H3章节 → 提取页面定位 → 推断规范类型 → 存入页面配置
每个H3章节 → 解析功能用例表 → 提取场景/前置条件/操作步骤/预期结果/异常分支 → 存入用例列表
每个H3章节 → 解析关键字段表 → 提取字段/从哪来/往哪去/变了怎么办 → 存入字段映射
```

### Phase 3: 全局流程提取

```
草案文档 → 检索跳转关系表 → 提取页面A/触发条件/页面B → 存入导航配置
草案文档 → 检索状态流转表 → 提取当前状态/操作/目标状态 → 存入状态配置
草案文档 → 检索登录拦截表 → 提取受保护页面列表 → 存入权限配置
```

## 状态标记解析规则

| 状态标记 | 解析规则 | 输出处理 |
|----------|----------|----------|
| `[草案]` | 提取预期结果中的 `[草案]` 标记 | 正常生成，标记来源为草案 |
| `[待原型]` | 提取页面标题中的 `[待原型]` 标记 | 生成后附加"待验证"标注 |
| `[待确认]` | 提取字段来源/业务问题中的标记 | 生成后附加确认提示区域 |
| `[必验]` | 提取假设清单中的 `[必验]` 标记 | 生成后附加验证清单区域 |
| `[风险]` | 提取假设清单中的 `[风险]` 标记 | 生成后附加风险提示区域 |
| `[暂略]` | 提取假设清单中的 `[暂略]` 标记 | 不生成对应部分，记录待补充 |

## 页面定位解析

### 页面定位格式

```markdown
> **页面定位**：{一句话：这个页面解决什么问题}
```

### 规范推断规则

从页面定位提取关键词，推断适用规范：

| 关键词 | 推断规范 |
|--------|----------|
| 运营人员、管理员、后台、审批、管理 | 后台规范 |
| 用户、小程序、移动端、下单、购物 | 小程序规范 |

## 功能用例表解析

### 表格结构（V0.22 六列格式）

```markdown
| # | 场景 | 前置条件 | 操作步骤 | 预期结果 | 异常分支 |
|---|------|----------|----------|----------|----------|
| 1 | {场景名} | {前置条件} | {操作描述} | {预期结果描述} `[草案]` | {异常} `[草案]` |
```

**兼容性：** 若草案仍为旧版 4 列格式（场景/操作/预期结果），前置条件置空、异常分支置空，不报错。

### 解析字段

| 字段 | 解析规则 | 输出用途 |
|------|----------|----------|
| 场景 | 提取场景名词短语 | 组件场景命名 |
| 前置条件 | 提取触发条件描述 | 条件渲染/禁用逻辑（详见 mappings/use-case-to-component.md） |
| 操作步骤 | 提取动词+元素描述 | 交互事件绑定 |
| 预期结果 | 提取结果描述 + 状态标记 | 组件渲染规则 |
| 异常分支 | 提取 `异常条件 → 处理方式` 格式 | 错误/空状态/校验组件生成 |

### 前置条件解析

| 前置条件模式 | UI 处理 |
|--------------|---------|
| `已登录` | 页面加载时检查登录态，未登录跳转登录页 |
| `已选商品` / `已填地址` | 提交按钮初始禁用，条件满足后启用 |
| `列表非空` / `有数据` | 渲染列表态，否则渲染空状态 |
| `进入商品详情` | 默认前置条件，无需特殊处理 |
| `—` | 无前置条件，正常渲染 |

### 异常分支解析

异常分支格式：`异常条件 → 处理方式`

| 异常分支模式 | UI 处理 |
|--------------|---------|
| `Toast提示 → ...` | 生成 Toast 组件，绑定到失败回调 |
| `弹窗 → ...` | 生成 Modal/Confirm 组件 |
| `置灰` / `禁用` | 元素添加 disabled 状态 |
| `跳转 → 页面名` | 生成 location.href 跳转 |
| `展示空状态` | 生成 EmptyState 组件 |
| `—` | 无异常分支，正常渲染 |

### 操作关键词提取

从操作步骤列提取关键动词：

| 关键词类型 | 示例 | 组件映射 |
|------------|------|----------|
| 点击操作 | 点击按钮、点击卡片 | onclick事件 |
| 输入操作 | 输入框、填写 | input组件 |
| 选择操作 | 选择、下拉、Tab切换 | select/tab组件 |
| 滑动操作 | 滑动、拖拽 | touch事件 |
| 查看操作 | 查看、显示 | 展示组件 |

## 关键字段数据来源解析（V0.22 四列溯源三问）

### 表格结构

```markdown
| 字段 | 从哪来 | 往哪去 | 变了怎么办 |
|------|--------|--------|------------|
| {字段名} | {来源描述} | {目标描述} `[待确认]` | {变更策略} `[草案]` |
```

**兼容性：** 若草案仍为旧版 2 列格式（字段/数据来源），将"数据来源"内容填入"从哪来"，"往哪去""变了怎么办"置空。

### 解析字段

| 字段 | 解析规则 | 输出用途 |
|------|----------|----------|
| 字段名 | 提取字段名称 | UI元素命名 |
| 从哪来 | 提取来源描述 + 状态标记 | UI元素可编辑性 + 数据绑定（详见 mappings/field-to-ui-element.md） |
| 往哪去 | 提取目标接口/表/流程 | 提交事件绑定 + 数据流向 |
| 变了怎么办 | 提取变更同步策略 | 校验规则 + 数据更新逻辑 |
| 状态标记 | 提取 `[待确认]` 标记 | 确认提示生成 |

### 从哪来分类（数据溯源 - 来源）

| 来源类型 | 关键词 | UI处理 |
|----------|--------|--------|
| 用户操作 | 用户操作、用户输入 | 输入组件（可编辑） |
| 系统生成 | 系统生成、自动记录 | 展示组件（不可编辑） |
| 后台查询 | 品牌商城后台、API | 动态加载组件 |
| 上一页传入 | 上一页面传入 | 参数传递处理 |
| 前端计算 | 前端实时计算 | 计算属性/派生展示 |
| 本地存储 | localStorage、sessionStorage | 本地读写 |

### 往哪去分类（数据溯源 - 去向）

| 去向类型 | 关键词 | UI处理 |
|----------|--------|--------|
| 提交接口 | 提交xxx API、POST | 表单提交事件绑定 |
| 写入表 | 写入xxx表 | 隐含在提交逻辑中 |
| 触发流程 | 触发xxx流程 | 提交后跳转/调用 |
| 仅展示 | 仅展示、不提交 | 纯展示组件，无提交 |

### 变了怎么办分类（数据溯源 - 变更）

| 变更策略 | 关键词 | UI处理 |
|----------|--------|--------|
| 实时同步 | 实时刷新、实时同步 | 数据变化触发重渲染 |
| 手动刷新 | 手动刷新 | 刷新按钮/下拉刷新 |
| 不影响已提交 | 不可变 | 提交后字段置只读 |
| 触发通知 | 触发通知 | 状态变更提示 |

## 跳转关系解析

### 表格结构

```markdown
| # | 页面A | 触发条件 | 页面B |
|---|-------|----------|-------|
| 1 | {页面名} `[待原型]` | {触发条件} | {页面名} `[待原型]` |
```

### 解析字段

| 字段 | 解析规则 | 输出用途 |
|------|----------|----------|
| 页面A | 提取源页面名称 | 跳转起点 |
| 触发条件 | 提取触发动作 | 跳转事件绑定 |
| 页面B | 提取目标页面名称 | 跳转目标URL |

## 状态流转解析

### 表格结构

```markdown
| 当前状态 | 可执行操作 | 目标状态 | 执行者 |
|----------|-----------|----------|--------|
| {状态A} `[草案]` | {操作} | {状态B} `[草案]` | {执行者} |
```

### 解析字段

| 字段 | 解析规则 | 输出用途 |
|------|----------|----------|
| 当前状态 | 提取状态名称 | 状态样式定义 |
| 可执行操作 | 提取操作名称 | 状态变更触发 |
| 目标状态 | 提取状态名称 | 状态流转目标 |
| 执行者 | 提取执行者角色 | 权限判断 |

## 错误处理

| 错误类型 | 处理方式 |
|----------|----------|
| 缺少HMW问题陈述 | 提示补充，使用默认页面定位 |
| 功能用例表缺失 | 提示补充，无法生成页面 |
| 关键字段缺失 | 提示补充，使用占位符生成 |
| 状态流转定义不清 | 提示补充，跳过状态逻辑生成 |

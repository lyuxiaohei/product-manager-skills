# 功能用例到组件映射规则 V2.0

定义如何将功能用例表中的每一行映射到具体的UI组件实例。

V2.0 变更：新增前置条件→条件渲染/禁用映射、异常分支→错误/空状态/校验组件映射（适配 logic-list-spec V0.22 六列用例表）。

---

## 映射原则

**一用例一组件**：每个功能用例对应至少一个UI组件实例。

**前置条件驱动渲染**：前置条件列决定组件的初始状态（可见/禁用/隐藏）。

**异常分支驱动边界**：异常分支列决定错误、空状态、校验失败等边界场景的组件。

## 前置条件映射（条件渲染/禁用）

前置条件列（V0.22 新增）决定组件的初始可用状态。映射为组件的 `visible`/`disabled`/`guard` 配置。

| 前置条件模式 | 映射组件 | 组件配置 |
|--------------|----------|----------|
| 已登录、已授权 | 页面守卫 | 加载时检查登录态，未登录跳转登录页 |
| 已选商品、已选规格 | 提交按钮 | 初始 `disabled: true`，条件满足后启用 |
| 已填地址、已填必填项 | 提交按钮 | 初始 `disabled: true`，表单完整后启用 |
| 已选择类型、已选择xx | 下游组件 | 依赖前置选择，未选时隐藏或置灰 |
| 列表非空、有数据 | 列表区 | 渲染列表态；否则渲染 EmptyState |
| 列表为空、无数据 | 空状态区 | 渲染 EmptyState；否则渲染列表 |
| 购物车有商品 | 结算按钮 | 有商品时显示，否则隐藏 |
| 进入xx页、页面加载完成 | — | 默认前置条件，无需特殊处理 |
| —（无前置条件） | — | 正常渲染，无 guard |

**映射输出格式（前置条件）：**

```json
{
  "component_type": "Button",
  "config": {
    "text": "立即支付",
    "guard": {
      "type": "disabled",
      "condition": "!selectedAddress || !selectedItems",
      "enable_message": "请选择收货地址"
    }
  }
}
```

## 异常分支映射（错误/空状态/校验组件）

异常分支列（V0.22 新增）格式为 `异常条件 → 处理方式`，映射为错误处理组件和逻辑。

| 异常分支模式 | 映射组件 | 组件配置 |
|--------------|----------|----------|
| `Toast → xxx` / `提示 → xxx` | `Toast` | message、duration、type(error/warning) |
| `弹窗 → xxx` / `Modal → xxx` | `Modal` | title、content、确认/取消 |
| `置灰` / `禁用` / `不可选` | 状态调整 | 元素 `disabled: true` + 置灰样式 |
| `跳转 → 页面名` | 跳转逻辑 | `location.href='xxx.html'` |
| `展示空状态` / `无数据` | `EmptyState` | 图标+文案+引导按钮 |
| `校验失败 → xxx` | 校验提示 | 表单错误提示、红框、错误文案 |
| `返回xxx` / `留在当前页` | 导航逻辑 | `history.back()` 或阻止跳转 |
| `刷新 → xxx` | 刷新逻辑 | 重新拉取数据/重渲染 |
| `重置 → xxx` | 重置逻辑 | 表单清空、状态还原 |
| —（无异常分支） | — | 正常渲染，无错误处理 |

**映射输出格式（异常分支）：**

```json
{
  "component_type": "Toast",
  "trigger": "submitFailed",
  "config": {
    "message": "库存不足",
    "type": "warning",
    "duration": 2000,
    "after": "refreshProductList"
  }
}
```

## 前置条件 × 异常分支 组合示例

一个完整的 V0.22 用例（6列）映射示意：

```markdown
| 3 | 提交订单 | 已选商品、已填地址 | 点击"立即支付" | 跳转支付页 | 库存不足→Toast"库存不足"并刷新 |
```

**完整映射输出：**

```json
[
  {
    "component_type": "Button",
    "component_name": "立即支付",
    "config": {
      "text": "立即支付",
      "type": "primary",
      "guard": {
        "type": "disabled",
        "condition": "!selectedAddress || cartItems.length === 0"
      },
      "onclick": "submitOrder"
    }
  },
  {
    "component_type": "Toast",
    "trigger": "STOCK_INSUFFICIENT",
    "config": {
      "message": "库存不足",
      "type": "warning",
      "after": "refreshCart"
    }
  }
]
```

## 映射流程

### Step 1: 操作列关键词提取

```
用例操作 → 分词 → 提取动词 + 名词 → 查询映射矩阵
```

### Step 2: 预期结果列元素识别

```
用例预期结果 → 识别UI元素关键词 → 查询映射矩阵
```

### Step 3: 前置条件列映射（V0.22）

```
前置条件 → 识别条件类型（登录态/数据状态/必填项）→ 生成guard配置
```

### Step 4: 异常分支列映射（V0.22）

```
异常分支 → 解析"异常条件 → 处理方式" → 生成错误处理组件
```

### Step 5: 组件配置生成

```
映射结果 → 提取具体参数 → 生成组件配置JSON（含guard和异常组件）
```

## 用例未映射处理

| 未映射情况 | 处理方式 |
|------------|----------|
| 操作关键词未匹配 | 使用基础 `Button` 组件，提示补充 |
| 预期结果未匹配 | 使用默认样式，提示补充 |
| 复合用例（多个操作） | 拆分为多个组件，顺序绑定事件 |

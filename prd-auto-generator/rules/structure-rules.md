# 文档结构规则 V7.0

定义版本PRD的章节编排：模块级 3 段、编号体系、必选/可选章节。

**V7.0 重构**（相对 V6.0）：页面级 6 节 → 模块级 3 段；引入 UC/AC/代号编号体系；可选章节 4 个 → 2 个；非功能需求并入核心流程。

## 章节结构总览

```
{项目名} PRD V{版本号}
├── 一、范围边界                    [必选]
│   ├── 1.1 本次做（In Scope，含代号列）
│   └── 1.2 本次不做（Out of Scope）
├── 二.1 角色列表                   [必选，含代号列]
│
├── 二.{代号}.2 功能用例与验收      [必选，每模块重复]
│   ├── 功能用例表（UC-{代号}-{n}）
│   ├── 验收标准（AC-{代号}-{n} + 对应用例）
│   └── 测试账号（全文仅首个模块）
├── 三.{代号} 核心流程              [必选，每模块重复]
│   ├── 3.{代号}.1 主流程（Mermaid）
│   ├── 3.{代号}.2 状态流转（含守卫条件）
│   └── 3.{代号}.3 异常与边界（功能异常 + 非功能）
├── 四.{代号} 数据流向              [必选，每模块重复]
│   ├── 4.{代号}.1 数据责任（角色×动作，中文动词）
│   └── 4.{代号}.2 字段说明（9列，首列字段中文名）
│
├── {N}、单据流转                   [可选 · 有核心实体时]
├── {N+1}、接口时序                 [可选 · 跨系统时]
└── {N+2}、实时通信                 [可选 · 有实时推送时]
```

## 必选章节

### 一、范围边界

放在文档最前，让读者 1 分钟内理解本次做什么。

- **1.1 In Scope**：`编号 / 模块 / 代号 / 功能点 / 优先级 / 说明`
- **1.2 Out of Scope**：列表形式，每项附原因

### 二.1 角色列表

`角色 / 代号 / 说明`。代号全局唯一，后续 UC/AC/CRUD 全部引用。

### 模块 3 段（二~四）

每个本次范围内的模块重复输出 3 段，使用 `templates/prd-module-section.md`。模块编排顺序见下文。

## 可选章节

| 章节 | 生成条件 | 来源 |
|------|----------|------|
| 单据流转 | 有核心业务实体（订单/审批/商品上架） | 业务逻辑清单的核心单据流转 |
| 接口时序 | 涉及跨系统交互（前端→网关→服务→DB/MQ） | 业务逻辑清单的系统交互时序 |
| 实时通信 | 有 WebSocket/长连接推送（IM/客服/协作） | 业务逻辑清单的系统交互时序（推送链路） |

>
> **非功能需求**：已并入模块级「三.{代号}.3 异常与边界」的非功能子表，不再独立成章。
>
> **数据模型**：数据结构并入「四.{代号} 数据流向」，不单独成章。

## 模块编排顺序

模块按**用户操作路径**排序，而非按文件名/层级：

### 推荐顺序

1. **账号与登录**（A）— 用户最先接触
2. **个人中心**（P）— 登录后首个管理页
3. **商品与浏览**（P/G）— 核心浏览路径
4. **购物车**（C）
5. **下单与支付**（O）— 转化路径
6. **订单管理/售后**（O）— 售后路径

### 编排原则

- 同一操作流程的模块相邻（登录+注册、下单+支付）
- 先基础后高级，先主流程后辅助
- 模块代号取英文名首字母，全局唯一、不冲突

## 编号规则

| 对象 | 格式 | 示例 |
|------|------|------|
| 章节号（版本级） | 中文数字 | 一、二、三 |
| 章节号（模块级） | `{版本级章节}.{代号}.{子节}` | 二.P.2、三.O.1 |
| 功能用例 | `UC-{代号}-{序号}` | UC-P-01 |
| 验收标准 | `AC-{代号}-{序号}` | AC-P-02 |
| 业务规则 | `RULE-{代号}-{序号}` | RULE-P-001 |

**约束**：
- 所有标题必须带编号
- 模块代号全局唯一（Product→P，Order→O，Auth→A）
- UC↔AC 通过验收表「对应用例」列绑定，一条 UC 可对多条 AC
- 同一规则只编号一次

## 与重量版的差异

| 维度 | V6.0（重量版） | V7.0（本版） |
|------|---------------|-------------|
| 组织 | 页面级 6 节 | 模块级 3 段 |
| 版本概述 | 5 子节 | 范围边界 + 角色列表 |
| 可选章节 | 角色/单据/接口时序/非功能（4个） | 单据/接口时序（2个） |
| 验收 | 无 | Given-When-Then + UC↔AC |

## 常见错误

| 错误 | 正确做法 |
|------|----------|
| 按文件名排序模块 | 按用户操作路径排序 |
| 验收标准不填「对应用例」 | 每条 AC 绑定到 UC |
| 测试账号每模块重复 | 全文仅首个模块输出，其余引用 |
| 统计指标单列章节 | 作为字段行写入字段说明表 |
| 简单模块硬凑满 3 段所有子节 | 按需省略，不留空占位 |
| 编号跨模块冲突 | 模块代号全局唯一 |
| 非功能需求单列章节 | 并入「异常与边界」非功能子表 |

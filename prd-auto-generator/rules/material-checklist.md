## 读取顺序

### 第一层：业务规格（必读）

| # | 材料 | 路径示例 | 作用 |
|---|------|----------|------|
| 1 | 业务逻辑清单 | `业务逻辑清单_V{版本}.md` | 场景、用例、字段溯源、角色、单据、时序、边界 |

### 第二层：源码实现（按模块读取）

仅读取本次版本范围内的页面源码：

| 类型 | 路径模式 | 作用 |
|------|----------|------|
| 页面HTML | `pages/{name}.html` | 结构、交互、字段、样式 |
| 公共JS | `js/common.js`、`js/auth.js` | 数据结构、登录态、工具函数 |

## 业务逻辑清单章节 → PRD 模块 3 段映射

业务逻辑清单 V0.22 的章节，按下表映射到模块级 3 段：

### 页面级章节

| 清单章节 | 清单格式 | PRD 目标 | 解析要点 |
|----------|----------|---------|----------|
| 功能用例表 | 6列（场景/前置/操作/预期/异常） | 二.{代号}.2 功能用例与验收 | 场景→UC场景；操作+预期→UC动作+预期；异常分支→AC 异常验收；前置条件→AC 的 Given |
| 关键字段数据来源 | 4列溯源三问（字段/从哪来/往哪去/变了怎么办） | 四.{代号}.2 字段说明 | 从哪来→来源列；往哪去→去向列；变了怎么办→三.{代号}.3 异常与边界 |
| 业务逻辑增强 | 2列（问题/确认结果） | RULE-{代号}-{n} 或 异常与边界 | 确认结果→业务规则编号条目 |

### 全局章节

| 清单章节 | PRD 目标 | 解析要点 |
|----------|---------|----------|
| 角色泳道图 | 二.1 角色列表 | 提取角色 + 代号；交互矩阵按需并入核心流程 |
| 核心单据流转 | {N}、单据流转（可选） | 有核心实体时直接映射生命周期表 |
| 系统交互时序 | {N+1}、接口时序（可选） | 同步→同步接口表；异步→异步流程表 |
| 系统交互时序（推送链路） | {N+2}、实时通信（可选） | WebSocket action 协议→Action 表；推送时机 |
| 边界条件清单 | 三.{代号}.3 异常与边界·非功能子表 | 并发/幂等→并发子表；一致性→一致性子表 |
| 状态流转 | 三.{代号}.2 状态流转 | 补「守卫条件」列 |
| 页面导航 | 并入各模块或省略 | 模块级不强制独立导航章；跨模块跳转写入相关模块 |

> **统计口径**（销量/好评率/折扣）：清单若含，作为字段行写入字段说明表，来源列填计算公式。
>
> **示例账号**：清单的示例账号 → 首个模块「功能用例与验收」验收表后的测试账号块（全文一次）。

## 信息整合规则

1. **业务逻辑清单优先** — 源码与清单冲突时以清单为准，标注差异
2. **源码补充细节** — 清单未提但源码已实现的，纳入 PRD
3. **版本标记对齐** — PRD 模块范围与 index 版本标记一致
4. **数据来源标注** — 涉及商品数据时注明来源（后台/配置中心）

## 增量版本处理

生成新版本PRD且已有旧版时：

| 类型 | 判断 | PRD 处理 |
|------|------|---------|
| 新增模块 | 旧版无对应 | 完整生成 3 段 |
| 变更模块 | 源码或清单有调整 | 标注变更点，更新受影响子节 |
| 未变更模块 | 均无变化 | 引用旧版，标注「同 V{上一版本}」 |

变更标注：

```
## 二.O.2 功能用例与验收

> ⚠ V0.2 变更：新增「拼团下单」用例
```

编号衔接：旧版 UC/AC 编号保留不变，新增的续编（UC-O-06...）。

## 常见错误

| 错误 | 正确做法 |
|------|----------|
| 只读源码不读清单 | 先读清单建立业务认知，再读源码补充 |
| 把页面级清单章节硬塞进模块 | 按映射表归类到模块 3 段 |
| 忽略清单的角色泳道图 | 映射到二.1 角色列表 |
| 忽略清单的边界条件 | 映射到异常与边界的非功能子表 |
| 跳过现有PRD | 先读旧版，避免重复/矛盾 |
